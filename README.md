# interview-frontend
智能模拟面试平台前端

## 最近更新

### 2025-10-18
- **修复 Next.js 构建错误**: 解决服务器端组件调用客户端API的问题
  - **问题背景**：在 `/banks` 和 `/start` 页面构建时出现 "Attempted to call error() from the server but error is on the client" 错误
  - **根本原因**：
    - 页面文件顶部使用了 `"use server"` 指令（实际上页面组件默认就是服务器组件，不需要显式声明）
    - 在服务器端代码的 catch 块中调用了 Ant Design 的 `message.error()`，这是客户端 API，不能在服务器端使用
  - **解决方案**：
    - 移除两个页面文件顶部的 `"use server"` 指令
    - 将 `message.error()` 替换为 `console.error()`，在服务器端输出错误日志
    - 在文件开头添加注释说明代码目的和实现方式
  - **修复文件**：
    - `src/app/banks/page.tsx` - 题库列表页面
    - `src/app/start/page.tsx` - 主页/起始页面
  - **影响**：现在项目可以正常构建和导出静态页面，支持部署到生产环境

### 2025-10-17
- **添加面试结束消息处理**: 优化面试完成后的用户体验
  - **问题背景**：后端在面试结束时发送 `DONE` 消息，但前端未处理，导致WebSocket直接关闭，用户不知道面试已结束
  - **解决方案**：
    - 添加对 `DONE` 消息类型的处理逻辑，当收到面试结束信号时：
      - 自动停止录屏（如果正在录制）
      - 延迟1秒等待录屏保存完成
      - 显示精美的面试结束对话框，展示最终得分
      - 提供返回面试中心的按钮
    - 使用 `useRef` 保存最新得分，避免闭包问题导致显示旧分数
    - 优化WebSocket关闭日志，区分正常关闭和异常断开
  - **用户体验改进**：
    - 面试结束时显示大号得分卡片，视觉效果更好
    - 显示后端返回的结束消息（如果有）
    - 如果有录制，提示视频已保存到下载文件夹
    - 点击"返回面试中心"按钮可查看面试历史
  - 文件位置：`src/app/interview/start/page.tsx`

- **修复WebSocket连接在录屏时断开的关键Bug**: 彻底解决录屏导致面试连接中断的问题
  - **根本原因**：useEffect依赖项变化导致清理函数被触发，在录屏启动时意外关闭WebSocket
  - **解决方案**：
    - 引入 `wsInitializedRef` 标记，确保WebSocket只初始化一次，避免重复连接
    - 将WebSocket清理逻辑拆分到独立的useEffect，使用空依赖数组，只在组件真正卸载时清理
    - 主连接useEffect的清理函数只清理定时器，不再关闭WebSocket
    - 移除其他地方重复的WebSocket清理代码，统一清理逻辑
  - **时序优化**：
    - 虚拟人选择 → 录屏设置（延迟500ms）→ WebSocket连接（延迟1000ms）→ 录屏启动（延迟1500ms）
    - 确保每个步骤有足够的时间完成，避免相互干扰
  - **调试增强**：添加详细的控制台日志，方便追踪连接状态和问题排查
  - 文件位置：`src/app/interview/start/page.tsx`

- **面试页面连接诊断优化**: 优化WebSocket连接失败和输入框禁用的用户体验
  - 修复了输入框过于严格的禁用条件，现在只需WebSocket连接即可输入（不再强制要求麦克风开启）
  - 优化了占位符提示文字，根据不同状态显示不同提示（等待连接/请先开启麦克风/输入你的回答）
  - 增强了WebSocket连接失败时的错误提示，明确告知用户需要检查的三个要点：
    1. 后端服务已启动
    2. 运行在 localhost:8811
    3. 已登录
  - 改进了token认证失败的日志输出，帮助开发者快速定位认证问题
  - 优化了语音按钮的禁用逻辑和提示信息，提供更清晰的操作指引
  - 文件位置：`src/app/interview/start/page.tsx`
